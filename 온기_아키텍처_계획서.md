# 🌤 온기 (Ongi) — 아키텍처 계획서

> **하루 두 번, 따뜻한 말 한마디**
> 위로·격려·응원 문장을 웹에서 확인하고, 이메일 구독으로 정기 수신하는 서비스

---

## 1. 프로젝트 개요

### 1.1 서비스 소개

| 항목 | 내용 |
|------|------|
| 서비스명 | 온기 (Ongi) |
| 핵심 가치 | 지친 일상 속 위로·격려·응원의 문장을 전달하는 감성 구독 서비스 |
| 타겟 사용자 | 일상에서 정서적 지지가 필요한 직장인, 취준생, 학생 등 |
| 핵심 기능 | ① 웹 문장 열람 ② 이메일 구독 (하루 2회) ③ AI 맞춤 문장 생성 |

### 1.2 기능 요구사항

**MVP (Phase 1)**
- 웹에서 카테고리별(위로/격려/응원/지지) 문장 랜덤 노출
- 이메일 구독 등록/해지
- Spring Batch 기반 하루 2회 이메일 발송 (아침 8시, 저녁 9시)
- 미리 준비된 문장 DB에서 랜덤 선택

**확장 (Phase 2)**
- Claude API 기반 AI 맞춤 문장 생성
- 구독자 선호 카테고리 설정 (구독 시 또는 마이페이지)
- 매직 링크 인증 → 문장 좋아요/저장 기능

**고도화 (Phase 3)**
- Kafka 기반 대량 이메일 비동기 처리
- 구독자 통계 대시보드 (Admin)
- 문장 공유 (카카오톡 공유 API, 링크 복사)
- PWA 지원 및 푸시 알림

### 2.2 기술 선정 근거 (심화)

**Spring Batch 6을 선택한 이유**

Spring Batch 6은 Spring Boot 4.x / Spring Framework 7 기반의 메이저 릴리스(2025.11 GA)로, 이전 버전 대비 핵심적인 개선이 이루어졌다.

첫째, **ChunkOrientedStep** — 기존 ChunkOrientedTasklet/TaskletStep을 대체하는 새로운 Chunk 처리 구현이 stable로 승격되었다. Retry는 Spring Retry 대신 Spring Framework 7 내장 retry 기능으로 전환되어 의존성이 줄고, SkipPolicy 기반의 Skip 처리가 더 직관적이 되었다.

둘째, **Local Chunking** — 동일 JVM 내에서 여러 스레드가 Chunk를 병렬 처리하는 기능이 추가되었다. 구독자 수가 늘어나도 멀티코어를 활용하여 발송 처리량을 높일 수 있다. 기존의 병렬 처리 모델(parallel iteration)이 producer-consumer 패턴으로 재설계되어 throttling/backpressure 문제가 해결되었다.

셋째, **Graceful Shutdown** — 배치 Job 실행 중 SIGTERM 시그널을 받으면 현재 실행 중인 Step을 안전하게 종료하고, 재시작 가능한 상태로 Job Repository를 갱신한다. 이메일 발송 중 서버 재배포 시 중복 발송이나 데이터 불일치를 방지할 수 있다.

넷째, **JFR(Java Flight Recorder) 관측성** — Micrometer 메트릭에 더해 JFR 이벤트로 Job/Step 실행, Item 읽기/쓰기, 트랜잭션 경계를 최소 오버헤드로 프로파일링할 수 있다.

다섯째, **인프라 설정 간소화** — `@EnableBatchProcessing`이 JDBC에 종속되지 않으며, `@EnableJdbcJobRepository` / `@EnableMongoJobRepository`로 저장소별 설정이 분리되었다. JobRepository가 JobExplorer를, JobOperator가 JobLauncher를 확장하여 별도 Bean 정의가 불필요해졌다.

**Redis Cache-Aside 패턴 적용**

문장 데이터는 자주 읽히지만 변경은 드물다. 매 요청마다 DB를 조회하면 불필요한 부하가 발생하므로, Redis에 카테고리별 문장 목록을 캐싱한다. TTL을 1시간으로 설정하고, 신규 문장 등록 시 캐시를 무효화(evict)하는 전략을 사용한다.

**Kafka 도입 시점 (Phase 3)**

초기에는 Spring Batch의 동기 처리로 충분하지만, 구독자 5만 명 이상이 되면 발송 지연이 발생할 수 있다. 이 시점에 Kafka를 도입하여 발송 요청을 토픽에 발행하고, 다수의 Consumer가 병렬 처리하는 구조로 전환한다.

### 3.2 이메일 발송 플로우 (상세)

```
[Spring Scheduler]
     │
     │  Cron: 0 0 8,21 * * *
     ▼
[Spring Batch 6 Job 실행]
     │
     ├─ ChunkOrientedStep: 이메일 발송 (Reader → Processor → Writer 통합)
     │   │
     │   ├─ Reader: JpaPagingItemReader
     │   │   └─ SELECT * FROM subscriber WHERE status = 'ACTIVE'
     │   │   └─ Page Size: 100
     │   │
     │   ├─ Processor: EmailContentProcessor
     │   │   ├─ Redis에서 오늘의 문장 조회 (Cache Hit)
     │   │   │   └─ Miss 시 → DB 조회 → Redis 저장 (TTL: 12h)
     │   │   ├─ 구독자 선호 카테고리 기반 문장 필터링
     │   │   └─ Thymeleaf 이메일 템플릿 렌더링
     │   │
     │   ├─ Writer: EmailSendWriter (100명/Chunk)
     │   │   ├─ Spring Mail → SMTP (Gmail / AWS SES)
     │   │   ├─ 발송 성공 → tb_send_history INSERT
     │   │   └─ 발송 실패 → RetryPolicy(maxAttempts=3) → SkipPolicy & 로깅
     │   │
     │   ├─ Fault Tolerance (Batch 6 방식)
     │   │   ├─ RetryPolicy: Spring Framework 7 내장 retry (Spring Retry 탈피)
     │   │   └─ SkipPolicy: LimitCheckingItemSkipPolicy(skipLimit=50)
     │   │
     │   └─ Local Chunking (Phase 2+)
     │       └─ 멀티스레드 병렬 Chunk 처리로 발송 처리량 스케일업
     │
     ├─ Tasklet Step: 발송 완료 리포트
     │   └─ 성공/실패 건수 → Prometheus Metric + JFR Event 기록
     │
     └─ Graceful Shutdown
         └─ SIGTERM 시 현재 Chunk 완료 후 안전 종료 → 재시작 가능 상태 보장
```

## 4. ERD (데이터베이스 설계)

### 4.1 테이블 설계

```sql
-- ─── 문장 테이블 ───
CREATE TABLE tb_quote (
    quote_id      BIGINT AUTO_INCREMENT PRIMARY KEY,
    content       TEXT NOT NULL COMMENT '문장 내용',
    category      VARCHAR(20) NOT NULL COMMENT '카테고리: COMFORT, CHEER, ENCOURAGE, SUPPORT',
    source_type   VARCHAR(10) NOT NULL DEFAULT 'MANUAL' COMMENT 'MANUAL | AI',
    like_count    INT DEFAULT 0 COMMENT '좋아요 수',
    is_active     BOOLEAN DEFAULT TRUE,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category_active (category, is_active),
    INDEX idx_source_type (source_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ─── 구독자 테이블 ───
CREATE TABLE tb_subscriber (
    subscriber_id   BIGINT AUTO_INCREMENT PRIMARY KEY,
    email           VARCHAR(255) NOT NULL UNIQUE COMMENT '이메일 주소',
    name            VARCHAR(50) NULL COMMENT '이름 (선택)',
    status          VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' COMMENT 'ACTIVE | PAUSED | UNSUBSCRIBED',
    preferred_cats  JSON NULL COMMENT '선호 카테고리 ["COMFORT","CHEER"]',
    verify_token    VARCHAR(100) NULL COMMENT '이메일 인증 토큰',
    verified_at     DATETIME NULL COMMENT '인증 완료 시간',
    subscribed_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    unsubscribed_at DATETIME NULL,
    INDEX idx_status (status),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ─── 발송 이력 테이블 ───
CREATE TABLE tb_send_history (
    history_id      BIGINT AUTO_INCREMENT PRIMARY KEY,
    subscriber_id   BIGINT NOT NULL,
    quote_id        BIGINT NOT NULL,
    send_type       VARCHAR(10) NOT NULL COMMENT 'MORNING | EVENING',
    send_status     VARCHAR(10) NOT NULL DEFAULT 'SUCCESS' COMMENT 'SUCCESS | FAILED | RETRY',
    sent_at         DATETIME DEFAULT CURRENT_TIMESTAMP,
    error_message   TEXT NULL,
    FOREIGN KEY (subscriber_id) REFERENCES tb_subscriber(subscriber_id),
    FOREIGN KEY (quote_id) REFERENCES tb_quote(quote_id),
    INDEX idx_subscriber_date (subscriber_id, sent_at),
    INDEX idx_send_type_date (send_type, sent_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ─── 문장 좋아요 테이블 (Phase 2: 매직 링크 인증 후 구독자 연결) ───
CREATE TABLE tb_quote_like (
    like_id         BIGINT AUTO_INCREMENT PRIMARY KEY,
    quote_id        BIGINT NOT NULL,
    subscriber_id   BIGINT NOT NULL COMMENT '인증된 구독자 ID',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (quote_id) REFERENCES tb_quote(quote_id),
    FOREIGN KEY (subscriber_id) REFERENCES tb_subscriber(subscriber_id),
    UNIQUE KEY uq_quote_subscriber (quote_id, subscriber_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ─── 저장한 문장 테이블 (Phase 2: 매직 링크 인증 후 구독자 연결) ───
CREATE TABLE tb_saved_quote (
    saved_id        BIGINT AUTO_INCREMENT PRIMARY KEY,
    subscriber_id   BIGINT NOT NULL COMMENT '인증된 구독자 ID',
    quote_id        BIGINT NOT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (subscriber_id) REFERENCES tb_subscriber(subscriber_id),
    FOREIGN KEY (quote_id) REFERENCES tb_quote(quote_id),
    UNIQUE KEY uq_sub_quote (subscriber_id, quote_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ─── Spring Batch 메타 테이블 (자동 생성) ───
-- BATCH_JOB_INSTANCE, BATCH_JOB_EXECUTION, BATCH_STEP_EXECUTION 등
```

## 5. API 설계

### 5.1 REST API 명세

**문장 (Quote)**

| Method | Endpoint | 설명 |
|--------|----------|------|
| `GET` | `/api/v1/quotes/today` | 오늘의 랜덤 문장 조회 |
| `GET` | `/api/v1/quotes?category={cat}&page={p}&size={s}` | 카테고리별 문장 목록 (갤러리) |
| `POST` | `/api/v1/quotes/{id}/like` | 문장 좋아요 토글 |
| `POST` | `/api/v1/quotes/ai-generate` | AI 맞춤 문장 생성 요청 |

**구독 (Subscribe)**

| Method | Endpoint | 설명 |
|--------|----------|------|
| `POST` | `/api/v1/subscribers` | 이메일 구독 등록 |
| `GET` | `/api/v1/subscribers/verify?token={token}` | 이메일 인증 확인 |
| `PATCH` | `/api/v1/subscribers/{email}/preferences` | 선호 카테고리 변경 |
| `DELETE` | `/api/v1/subscribers/{email}` | 구독 해지 |

**구독자 인증 · 저장 (Phase 2)**

| Method | Endpoint | 설명 |
|--------|----------|------|
| `POST` | `/api/v1/auth/magic-link` | 매직 링크 발송 (이메일 입력) |
| `GET` | `/api/v1/auth/magic-link/verify?token={token}` | 매직 링크 인증 → JWT 발급 |
| `GET` | `/api/v1/subscribers/me/saved-quotes` | 저장한 문장 목록 |
| `POST` | `/api/v1/subscribers/me/saved-quotes/{quoteId}` | 문장 저장 |
| `DELETE` | `/api/v1/subscribers/me/saved-quotes/{quoteId}` | 저장 취소 |

### 5.2 주요 API Request/Response 예시

**구독 등록**

```json
  "category": "COMFORT",
```

## 6. 핵심 컴포넌트 설계

### 6.1 이메일 발송 Batch Job 구조

```
EmailSendJobConfig
├── Job: emailSendJob
│   ├── Step: morningEmailStep (아침 8시) — ChunkOrientedStep (Batch 6)
│   │   ├── Reader:  JpaPagingItemReader<Subscriber>
│   │   │            - pageSize: 100
│   │   │            - query: status = 'ACTIVE' AND verified_at IS NOT NULL
│   │   ├── Processor: EmailContentProcessor
│   │   │            - Redis에서 오늘의 문장 조회
│   │   │            - 선호 카테고리 필터링
│   │   │            - Thymeleaf 템플릿 렌더링 → MimeMessage
│   │   └── Writer:  EmailSendWriter
│   │                - JavaMailSender.send(message)
│   │                - 성공/실패 로깅
│   │                - chunkSize: 100
│   │                - RetryPolicy (Spring Framework 7 내장): maxAttempts=3
│   │                - SkipPolicy: LimitCheckingItemSkipPolicy(skipLimit=50)
│   │                - Local Chunking: 멀티스레드 병렬 Chunk 처리 (Phase 2+)
│   │
│   └── Step: sendReportStep
│       └── Tasklet: 발송 결과 집계 → Prometheus Metric + JFR Event 기록
│
├── Graceful Shutdown Hook
│   └── SIGTERM 수신 시 현재 Step 안전 종료 → Job Repository 상태 갱신 → 재시작 가능
│
└── Scheduler
    └── @Scheduled(cron = "0 0 8,21 * * *", zone = "Asia/Seoul")
```

### 6.2 Batch 6 Step 구성 코드 (ChunkOrientedStepBuilder)

```java
@Configuration
@EnableJdbcJobRepository  // Batch 6: JDBC 전용 어노테이션 분리
public class EmailSendJobConfig {

    @Bean
    public Job emailSendJob(JobRepository jobRepository, Step emailSendStep, Step reportStep) {
        return new JobBuilder("emailSendJob", jobRepository)
                .start(emailSendStep)
                .next(reportStep)
                .build();
    }

    @Bean
    public Step emailSendStep(JobRepository jobRepository,
                              JdbcTransactionManager transactionManager,
                              ItemReader<Subscriber> subscriberReader,
                              ItemProcessor<Subscriber, MimeMessage> emailProcessor,
                              ItemWriter<MimeMessage> emailWriter) {

        int chunkSize = 100;

        // Batch 6: RetryPolicy — Spring Framework 7 내장 retry 사용
        RetryPolicy retryPolicy = RetryPolicy.builder()
                .maxAttempts(3)
                .includes(Set.of(MailSendException.class, TransientException.class))
                .build();

        // Batch 6: SkipPolicy
        SkipPolicy skipPolicy = new LimitCheckingItemSkipPolicy(
                50, Set.of(MailSendException.class));

        // Batch 6: ChunkOrientedStepBuilder (TaskletStep 대체)
        return new ChunkOrientedStepBuilder<Subscriber, MimeMessage>(
                    jobRepository, transactionManager, chunkSize)
                .reader(subscriberReader)
                .processor(emailProcessor)
                .writer(emailWriter)
                .faultTolerant()
                .retryPolicy(retryPolicy)
                .skipPolicy(skipPolicy)
                .build();
    }
}
```

```java
@Component
public class ClaudeApiClient {

    @Value("${claude.api.key}")
    private String apiKey;

    private final WebClient webClient;

    public String generateComfortMessage(String situation, String category) {
        String prompt = buildPrompt(situation, category);

        ClaudeResponse response = webClient.post()
            .uri("https://api.anthropic.com/v1/messages")
            .header("x-api-key", apiKey)
            .header("anthropic-version", "2023-06-01")
            .bodyValue(Map.of(

                "max_tokens", 300,
                "messages", List.of(Map.of("role", "user", "content", prompt))

            .retrieve()
            .bodyToMono(ClaudeResponse.class)
            .block();

        return extractText(response);
    }

    private String buildPrompt(String situation, String category) {
        return """
            당신은 따뜻한 위로와 격려의 문장을 만드는 전문가입니다.
            카테고리: %s
            상황: %s
            규칙: 2~3문장, 진심 어린 표현, 한국어, 문장만 출력
```

6. **Claude API 연동** — 외부 AI API를 활용한 실시간 콘텐츠 생성
7. **이메일 인증 플로우** — Double Opt-In 구현으로 보안/스팸 방지
